## 習得した技術・概念

### 1. Tauriとは何か

- **What（何をするものか）**: React（画面）+ Rust（OS機能）を組み合わせてWindowsデスクトップアプリを作るフレームワーク
- **Why（なぜ必要か）**: AIカンパニーは「ローカル完結・BYOK」が原則のため、ブラウザアプリ（Next.js等）ではなくデスクトップアプリが必要。Electronと比べてアプリサイズが小さく、セキュリティモデルが強固。
- **Where（どこで使うか）**: SQLiteへのファイルアクセス、APIキーのOS安全保管（Windows DPAPI）、ファイルエクスポート（議事録MD出力）など「OSに近い処理」全般
- **身近な例えでの理解**:
  > Tauriは「Web画面（React）」という見た目の部分と、「工場の機械（Rust）」というOS操作部分を組み合わせたハイブリッドな仕組み。WebブラウザではできないファイルへのアクセスやOSのキーチェーン（金庫）を操作できるのがTauriの強み。

### 2. Rust開発環境のインストール要件（Windows）

- **What**: `rustc`（コンパイラ）+ `cargo`（パッケージマネージャ）+ Visual Studio Build Tools（C++ビルドツール）
- **Why**: RustはWindowsネイティブのアプリをビルドするためにMicrosoftのC++コンパイラを内部で利用する
- **インストール順序**: Visual Studio Build Tools → rustup-init.exe（順序を間違えると失敗することがある）
- **注意**: インストール後はVS CodeとTerminalを完全に再起動してPATHを反映させる必要がある

### 3. PowerShell vs bash のコマンド構文の違い

| 操作 | bash（Mac/Linux） | PowerShell（Windows） |
|---|---|---|
| コマンドの連結 | `cmd1 && cmd2` | `cmd1; cmd2` |
| 環境変数の参照 | `$VAR` | `$env:VAR` |
| ファイル一覧 | `ls` | `Get-ChildItem` または `ls` |

- **身近な例えでの理解**:
  > bashとPowerShellは「日本語と英語」のような関係。似た処理をしたくても、表現（構文）が違う。Windowsで開発するときは「PowerShell語」で話す必要がある。

### 4. .gitignore と .gitattributes の役割

- **.gitignore**: 「Gitに追跡させないファイル」を指定する設定。`node_modules/`（巨大な依存パッケージ群）や`target/`（Rustのビルド成果物）を除外することでリポジトリの肥大化を防ぐ。
- **.gitattributes**: 「ファイルの改行コードのルール」を設定する。WindowsはCRLF、Mac/LinuxはLFという違いが原因で差分ノイズが出ることを防ぐ。
- **身近な例えでの理解**:
  > .gitignoreは「会社に持ち込まない私物リスト」。.gitattributesは「社内の書式統一ルール（改行は必ずXX形式で）」。

## AIカンパニー固有の設計理解

### ROADMAPの現在地
```
ノードA（DBスキーマ）→ ノードB（4層マージ） → ... → ノードH（Markdownエクスポート）
^
│ ← ここから着手（次回セッション）
```

### 4層マージの核心
```
最終システムプロンプト =
  users.core_profile（ユーザーがAI社員全員に伝えること）
  + projects.values（このプロジェクトの判断軸・価値観）
  + departments.department_prompt（部署の思考の癖）
  + members.personality_prompt（個人の人格・専門性）
```
この構造がSQLiteのテーブル設計に直結しており、途中変更は全面改修になる根幹部分。

### 3層構造
```
AIカンパニー（アプリ）
└── プロジェクト（会社）← projects テーブル
    └── 部署（部門）← departments テーブル
        └── メンバー（AI社員）← members テーブル
```

## 遭遇したエラーと解決策

| 事象 | 原因 | 対策 |
|---|---|---|
| `rustc: コマンドが見つかりません` | Rustが未インストール | rustup-init.exeをrust-lang.org公式から取得しインストール（Visual Studio Build Toolsも同時に必要） |
| PowerShellで `&&` が使えない | PowerShellはbash構文の`&&`を解釈できない | `;` で区切るか、コマンドを別行に分けて実行 |
| Master_Handoff_Prompt.md が空に見えた | ファイル保存のタイミング問題（セッション開始直後にキャッシュが古かった可能性） | 再読み込みで解決。今後はユーザーが「ファイルに中身がある」と言ったら素直に再読み込みする |
