## 習得した技術・概念

### 1. Strangler Fig パターン（段階的置換パターン）
- **What**: 稼働中の巨大な既存コード（直書き fetch）を一度に全破壊して書き換えるのではなく、共通の薄い抽象インターフェース（`llmProvider.ts`）を挟み、内部実装だけを徐々に LangChain.js 等に差し替えていくリファクタリング手法。
- **Why**: 一括置換による大規模障害や動作停止のリスクを完全に回避し、型安全性とビルドのグリーン状態を維持しながら安全に移行するため。
- **Where**: `MeetingScreen.tsx` や `ChatScreen.tsx` の API コール部。
- **身近な例えでの理解**:
  > **「新旧水道管のバイパス工事」**: 家の中の蛇口（画面UI）や配管全体を一気に壊すのではなく、まず屋外の給水元栓に万能アダプター（`llmProvider.ts`）を取り付け、裏の給水管（fetch → LangChain）を少しずつ安全につなぎ替える作業。家の中の人は水道の使い方を一切変えずに、裏のインフラだけが最新化される。

### 2. 環境非互換ライブラリのカプセル化・ハイブリッドパターン
- **What**: 特定のプロバイダーSDK（`@langchain/anthropic`）が Node.js コアモジュール依存によって Tauri/Vite ブラウザ環境でビルド不能な場合、無理にポリフィル（偽の Node.js 模倣）を通さず、Rest API 自前 fetch をモジュール内部にカプセル化して併用する技術。
- **Why**: 外部ライブラリのバグやビルド崩壊のリスクからアプリ全体を保護し、安定したネイティブ Web API（`fetch`）の信頼性を活かすため。
- **Where**: `src/lib/llmProvider.ts` 内の `callAnthropicWithFetch()`。
- **身近な例えでの理解**:
  > **「海外家電のマルチプラグアダプター」**: 基本は世界共通規格のプラグ（LangChain）で揃えつつ、特殊な海外規格の家電（Anthropic）だけは専用の小さな変換プラグを内部に仕込み、外からは全て同じコンセント穴に見せる仕組み。

### 3. Fallback Chain パターン（自動迂回呼び出し）
- **What**: APIキー未設定や特定プロバイダーの通信障害が発生した際、画面側のコードに try-catch ループを書かせることなく、プロバイダー層（`llmProvider`）側で安全に次の利用可能なプロバイダーへ迂回試行する設計。
- **Why**: 画面コンポーネント（`MeetingScreen.tsx`）から約100行に及ぶプロバイダー試行ループを排除し、UI層とロジック層を完全に分離するため。
- **Where**: `src/lib/llmProvider.ts` 内の `callLLMWithFallback()` および `MeetingScreen.tsx` の `handleGenerateSummary()`。

## 遭遇したエラーと解決策

| 事象 | 原因 | 対策 |
|---|---|---|
| `@langchain/anthropic` の導入試行時にビルド/バンドル警告・エラー | パッケージ内部で `node:fs` や `node:crypto` 等の Node.js コアライブラリを直接要求するため、Tauri のフロントエンド環境（ブラウザ環境）と非互換 | パッケージの無理な導入を断念し、`https://api.anthropic.com/v1/messages` への自前 `fetch` 処理（`callAnthropicWithFetch`）を `llmProvider.ts` 内にカプセル化 |
| `MeetingScreen.tsx` の議事録サマリー生成処理のみ生 `fetch` が直書きで残存 | 会議中の通常発言と異なり、複数プロバイダーを順次試す「フォールバック制御」が含まれていたため置換がスキップされていた | `llmProvider.ts` に `callLLMWithFallback()` を新規追加し、サマリー生成処理も共通関数1行で呼び出す形にリファクタリング |
| `extractTokens(response as any)` に残っていた `any` 型 | LangChain の `AIMessage` 型のバージョン差分を回避するための簡易キャスト | `extractTokens(response: unknown)` とし、オブジェクト判定を行う型ガードを導入して `any` を完全排除 |
