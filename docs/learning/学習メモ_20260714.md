## 習得した技術・概念
- **What（何をするものか）**: CSS Flexboxを活用したスクロール領域と固定フッターの分離
- **Why（なぜ必要か）**: Tailwind CSSの `absolute` 配置は親要素の高さを基準にするため、内容が動的に変わるSPAでは他の要素（カード等）と重なる事故が起きやすい。確実に下部に固定するため。
- **Where（どこで使うか）**: 画面下部に常に表示させたいCTA（Call To Action）ボタンやフッター領域。
- **身近な例えでの理解**: 
  > 【お弁当箱の仕切り】
  > absoluteは「おかずの上に無理やりシールを貼る」ようなもので、中身が増えると被ってしまいます。一方flex-colの固定領域は「最初からおにぎり専用の仕切り箱を用意する」ようなもので、他のおかずがどれだけ増えても絶対に干渉しません。

## 遭遇したエラーと解決策
| 事象 | 原因 | 対策 |
|---|---|---|
| プロジェクト作成時にDBエラー | SQLiteの外部キー制約違反（IDの取り回しミス） | Tauri SQLプラグインの `execute` 戻り値から `lastInsertId` を取得し、後続のINSERT文（部署・メンバー）で正確に使用するよう修正 |
| 「会議を始める」ボタンがカードの上に被る | `absolute bottom-6` の親要素の高さ計算が、動的グリッド要素と合っていなかった | 全体を `flex flex-col` にし、スクロール領域と固定フッターバー領域を完全に分割。ボタンから `absolute` を外した |
| アバター画像が洋ゲー風で世界観に合わない | DALL-E等の画像生成プロンプトの指定不足（「ピクセルアート」だけだと陰影がリアルになる） | `no realistic shading`、`flat cute anime style`、`Harvest Moon style` 等のキーワードを追加し、日本的な可愛いドット絵に方向修正 |
