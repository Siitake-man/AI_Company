## 🛠️ 具体的な改修対象と詳細指示

### 1. APIKeySetupScreen.tsx (S1: オンボーディング画面)
- **参考モック**: `screen_S1_onboarding.png`
- **方針**: 画面中央に丸みのあるウッドボーダーで囲まれた「Welcome to AI Team Builder 🍃」パネルを配置し、APIキーの入力枠と保存アクションを可愛くリデザインしてください。

### 2. CreateProjectScreen.tsx (S3: 新規プロジェクト作成画面)
- **参考モック**: `screen_S3_new_project.png`
- **方針**: プロジェクトの目的や価値観の入力フォームを「タイプライター風の入力欄（インナーシャドウ付き）」にし、部署のチェックボックスをはんこスタンプ調（枠線を二重にしたり、赤みがかったスタンプ枠風）にしてください。

### 3. TeamManageScreen.tsx (S4/S5: メンバー管理とエディタ)
- **参考モック**: `screen_S4_team.png` / `screen_S5_member_editor.png`
- **方針**: 
  - メンバーの縦並びリストを、手帳のルーズリーフインデックスカード風（役割別カラー）にデザインしてください。
  - メンバー編集時に開くポップアップ（またはモーダル）は、モックS5のような「クラシックな手書きノート帳（横の罫線付き）」のモーダルにしてください。

### 4. ChatScreen.tsx (S6: 1on1チャット画面)
- **参考モック**: `screen_S6_chat.png`
- **方針**: 
  - チャットのタイムラインにある発言の吹き出しを、**「少し傾いて貼り付けられた付箋紙（ポストイット）風」**（`transform: rotate(-1deg)` や `box-shadow`）にブラッシュアップしてください。
  - 下部のメッセージ入力欄と送信ボタンを、ノートのマージン罫線と黄色いぷっくりボタンにリデザインしてください。

### 5. MeetingScreen.tsx (S7: 会議画面)
- **参考モック**: `screen_S7_meeting.png`
- **方針**:
  - 現在進行しているLLM発言ループ（2.5秒間隔）やアジェンダはそのまま維持し、アジェンダ枠をウッドパネルに、発言ログをレトロなレターカード風の装飾に変更してください。
  - 左サイドバーの「発言中のメンバー」がゴールドの枠線とパルスシャドウで際立つようにハイライトしてください。
  - 下部のコントロールボタン（一時停止、スキップ、終了）を、テープレコーダーの物理ボタン風（押し込みの効くセカンダリボタン）にリデザインしてください。

### 6. 会議サマリー表示 (S8: 議事録・サマリー画面)
- **参考モック**: `screen_S8_summary.png`
- **方針**: 
  - 会議終了時に生成される要約サマリー（主要論点、PRO/CON、決定事項、ToDo）を、ルーズリーフ用紙風に綺麗にブロック整理して表示してください。

  - テストケース（ケース1: 鈴木さん[ID: 1]、ケース2: 悪魔の代弁者[ID: 2]）の選択変更をトリガーに、`src/lib/promptMerger.ts` の `getMergedSystemPrompt` を非同期で呼び出し、ユーザープロフィール（第1層）〜個人人格（第4層）が正しくマージされた結果を取得して、画面内のテキストエリア（`mergedPrompt`）に動的に表示するリアクティブな連動ロジックを実装してください。

### 2. H8: 議事録サマリーのMarkdownエクスポート実ロジックの実装
- **現状**: 会議サマリー（S8）の「Export to Markdown」ボタンを押した際の動作が未実装です。
- **指示**:
  - ボタンがクリックされた際、Tauriのダイアログプラグイン（`@tauri-apps/plugin-dialog` の `save` または `@tauri-apps/api/dialog`）を呼び出し、保存ファイル名指定ダイアログを起動してください。
  - 指定された保存先に対して、TauriのFSプラグイン（`@tauri-apps/plugin-fs` の `writeTextFile` または `@tauri-apps/api/fs`）を使用し、会議の要約やログ全体をマークダウン形式でローカルディスクに物理的にエクスポート・保存する処理を完成させてください。

### 3. 会議進行時の未設定APIキーに対する安全ガードの堅牢化
- **現状**: `MeetingScreen.tsx` の議論進行ループ中、モデルのAPIキーが未設定の場合は警告を出してスキップするだけで、ループは回り続けます。
- **指示**:
  - APIキーが未設定の場合は、単にスキップして次のスピーカーに渡してループを流すのではなく、会議進行自体を一時停止（`isPaused = true`）にし、ユーザーが気付きやすい警告モーダルまたはシステム通知ダイアログを表示して安全に停止するようにしてください。

## 🚫 作業上の制約 (絶対に遵守すること)

1. **ロジックの破壊は厳禁**:
   - 会議の自動タイマーループ (`useEffect` 内の 2.5秒間隔進行)、LLM API呼び出し、アジェンダ注入、料金計算 (`calculateCost`)、SQLiteへの保存（`meetings`, `meeting_summaries`, `member_learnings`）のロジックコード自体は絶対に修正しないでください。
   - `App.tsx` から渡されている各種 props やハンドラ（`setCurrentScreen`, `setChatMemberId` 等）の整合性を崩さないでください。
2. **既存データ定義・構造の破壊禁止**:
   - SQLiteテーブル定義や、既存のテーブル名・カラム名を無断で変更しないでください。
3. **Rustコード (Tauri API) は触らない**:
   - 修正はフロントエンド（`src/**/*.{ts,tsx}` および `index.css`）のみで完結させてください。今回もTauriのフロントエンドJavaScript/TypeScript APIのみをインポートして呼び出し、Rust側のコードは編集しないでください。
4. **省略コメント禁止**:
   - コードを出力・書き換える際は、`// ...既存コードそのまま` などの手抜き省略表現をせず、プロダクション品質で安全に出力してください。
